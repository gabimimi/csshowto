CSS Instructions (6.100A style)

You will edit styles.css to style the Headline Analyzer. The HTML/JS already exists. Your job is to make the UI look clean, readable, and interactive—without breaking the app.

What CSS controls

  • Layout: where things sit on the page (centering, spacing, grids).
  • Look: colors, shadows, borders, typography.
  • Feedback: hover effects, focus rings, animations (bar growth, pop-in).

Project goal

When the user clicks analyze, the results appear with a nice animation, the progress bar grows smoothly, and the colors match the risk label.

Your file has “Tasks”

Inside styles.css you’ll see comments like (Task 1.2). Follow them in order. You can be creative, but don’t break the contracts.

The CSS Contract (do NOT break this)

Rule #1: do not rename IDs/classes in HTML

JavaScript finds elements by ID and updates them. If IDs/classes change, the site “stops working”. CSS also depends on classes like .card, .metrics, .bar.

Selectors you must keep

  • Layout: body, .container, .card, .row, .hidden
  • Input bar: .chatBar, #headline, #analyzeBtn, #clearBtn
  • Results: #result, .resultTop, .label, .barWrap, .bar
  • Metrics: .metrics, .metric, .k, .v
  • (Optional rings): .ring, .ring__inner, .vwrap, .pts

CSS Theory you need (everything beyond “basic loops”)

1) Selectors (how CSS targets elements)

  • h1 targets all h1 tags.
  • .card targets anything with class="card".
  • #headline targets the element with id="headline" (IDs are unique).
  • .chatBar textarea#headline targets the headline textarea only when inside .chatBar.

2) Box model (spacing)

  • padding = inside spacing.
  • border = outline thickness.
  • margin = outside spacing.
  • box-sizing: border-box makes widths predictable.

3) Flexbox (layout)

  • display:flex makes children align in a row by default.
  • flex-direction: column stacks children vertically.
  • justify-content aligns on main axis.
  • align-items aligns on cross axis.
  • gap creates spacing between children.

4) Transitions vs keyframe animations

  • Transition: animates when a property changes (ex: width changes).
  • Keyframes: runs a timeline you define (@keyframes).
/* transition example */
.bar { transition: width 700ms ease; }

/* animation example */
@keyframes popIn { from{opacity:0} to{opacity:1} }

CSS Tasks (mapped to your Task comments)

Work inside styles.css. Follow these tasks in order. You can change values freely (sizes, spacing, colors), but keep the selectors the same. :contentReference[oaicite:1]{index=1}

Task 1 — Make the background cover the entire screen

In your file, Task 1 is inside the body rule (Task 1.1–1.3). Goal: The background should be the background of the whole screen, not just behind the container.

What to do
  • Task 1.1 (font-family): choose a font stack you like (Google-ish tends to be clean and simple).
  • Task 1.2 (background): pick a background color/gradient. Keep background-size: cover and no-repeat.
  • Task 1.3 (centering with flex): use flex to center the page content in a column.

Important: don’t cut off the results

If you vertically center too aggressively (like justify-content:center), the results card can push the title off-screen. A safe pattern is: justify-content: flex-start plus good padding on .container.

Task 2 — Titles and subtitle typography

These are h1 and .subtitle.

What to do
  • Task 2.1: style h1 (size, weight, letter spacing, alignment).
  • Task 2.2: style .subtitle (color should be muted, not full white).

Design principle

  • Titles: large, bold, tight letter spacing looks modern.
  • Subtitle: smaller and muted so it doesn’t compete with the title.

Task 3 — The chat/search bar (textarea + buttons on one line)

This is the “WhatsApp-style” input bar. You already have selectors like .chatBar and .chatBar textarea#headline in your CSS.

What to do
  • Make the container pill-shaped with border-radius: 999px.
  • Use :focus-within to show a focus ring when the user clicks inside.
  • Make the textarea transparent so it feels like one unified bar.
  • Make buttons circular and icon-sized (width/height equal).

Theory: :focus-within

.chatBar:focus-within means: “apply these styles if any child element is focused”. That lets you highlight the whole bar when the textarea is active.

Task 4 — Hidden elements should disappear

Your results card starts as class="card hidden". JS removes hidden when data arrives. CSS must ensure hidden elements are not visible.

What to do
  • Create (or keep) .hidden { display: none; }
  • If results show too early, you probably broke this rule.

Task 5 — Results header layout (.resultTop)

The top of results has the label chip, the numeric score, and the bar. It must wrap nicely on small screens.

What to do
  • Use display:flex, align-items:center.
  • Use justify-content: space-between to spread content.
  • Use flex-wrap: wrap to avoid overflow on mobile.

Task 6 — Label chip styling (.label)

This chip shows risk level (green/yellow/red).

What to do
  • Keep the chip pill style (rounded, padded).
  • Decide whether label is solid or tinted.
  • If you want maximum clarity: use solid background and white text.

Google yellow requirement

Google yellow is #fbbc04 (your CSS variable --g-yellow). If you use transparency, it might look like a “watermark”.

Task 7 — The main score bar (.bar)

This is the most “visual” part: the bar should grow to its score and show the score text inside. Your CSS already uses transition: width and shows text via ::after. :contentReference[oaicite:2]{index=2}

What to do
  • Ensure .bar starts with width: 0% in CSS.
  • Ensure it has transition: width ....
  • Ensure it has a background color for each label (.bar.green, etc.).
  • Ensure the text is white and centered.

If the bar “doesn’t grow”, here are the usual causes

  • No transition: width jumps instantly.
  • Not starting at 0%: browser has nothing to animate from.
  • Element hidden: if you set width while hidden, some browsers won’t animate well.

Task 8 — Headings and lists in results

This is where you make the content readable: section headers (“Why”, “Metrics”, “Tips”) and list font sizes. In your file, these are Task 8.1–8.3.

What to do
  • Task 8.1: style h2 (size/weight) for the label header.
  • Task 8.2: style h3 for section headings.
  • Task 8.3: style ul, ol for comfortable reading.

Design principle: hierarchy

  • h2 should be bigger than h3.
  • Lists should be “body text size” (not tiny).

Task 9 — Metric cards layout (.metric)

Metrics are small “key/value” cards. Make them evenly spaced and easy to scan.

What to do
  • Use padding for breathing room.
  • Use justify-content: space-between to separate key/value.
  • Use gap so text doesn’t collide.

Task 10 (Optional/Advanced) — Circular rings for metric “weight”

This is the “extra feedback”: each metric can show a circular progress ring. Your CSS includes a ring system using conic-gradient and an animation that fills it. :contentReference[oaicite:3]{index=3}

How it works (concept)
  • The ring is a circle whose background is a conic-gradient.
  • The gradient angle depends on --p (a percentage number).
  • Animation changes --p from 0 to --target.

Theory: conic-gradient

A conic gradient draws a “pie chart” around a circle. If we fill 60%, it looks like a ring that is 60% complete.

Important note

Rings also require the HTML/JS to create ring elements and set --target. If your current JS does not build rings yet, you can keep this as “bonus styling” for later, or only focus on the main bar first.

Biggest success pattern: get the layout + bar working first (Tasks 1–7), then polish typography and metrics (Tasks 8–9), then add rings if time (Task 10).

Debugging checklist (CSS)

Fast workflow

  • Right click → Inspect (DevTools).
  • Click the element that looks wrong.
  • See which CSS rules apply and which are crossed out.
  • Try changing values in DevTools first, then copy into styles.css.

Common break #1: Title disappears

Cause: justify-content:center on body + content grows. Fix: use justify-content:flex-start and allow scrolling.

Common break #2: Bar has no color

Cause: missing .bar.green/.yellow/.red backgrounds, or JS isn’t assigning classes. Fix CSS first; then confirm in DevTools the bar element has class green or yellow.
